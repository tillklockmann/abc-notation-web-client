# notation

A local-first, browser-only music notation editor. You pick a folder on your own disk, and the app reads and writes `.notation` files there directly — no backend, no accounts, no sync. Sheet music is written as [ABC notation](https://abcnotation.com/) and rendered live to SVG with [abcjs](https://www.abcjs.net/).

The UI is in German.

## Requirements

- **Chrome or Edge.** The app depends on the File System Access API (`window.showDirectoryPicker`), which Firefox and Safari don't support. Unsupported browsers get a warning instead of a folder picker.
- **Internet connection at runtime.** abcjs is loaded from a CDN, not bundled.
- **Node.js** (for development only).

## Getting started

```sh
npm install
npm run dev        # Vite dev server at http://localhost:5173
```

Other scripts:

```sh
npm run build      # type-check (vue-tsc) + production build
npm run preview    # serve the production build
```

There is no test runner or linter; `npm run build` is the only check.

## Features

- **Folder-based library** — pick a project folder once; the handle persists across reloads (IndexedDB). Create, rename, and delete pieces from the offcanvas sidebar. An empty folder is seeded with an example piece.
- **Autosave** — changes are written to disk with a 400 ms debounce; a status pill shows the current save state.
- **Split view** — rendered sheet on the left, raw ABC text with a line-number gutter on the right, separated by a draggable splitter (ratio persisted). The text panel can be hidden entirely ("Volltext").
- **Editable page heading** — title and subtitle are edited directly on the rendered page.
- **Meter and key from the toolbar** — dropdowns for meter (4/4, 3/4, 2/4, 6/8, 12/8) and key (C, G, D, A, E, F, B♭, E♭, A♭, and common minor keys); the ABC body stays header-free.
- **Export** — download the piece as a single SVG, or print to PDF via the browser's print dialog. Print layout breaks pages *between* staff systems, never through them.
- **ABC warnings** — parse warnings are shown with line numbers remapped to the visible ABC text.

## The `.notation` format

A `.notation` file is a small frontmatter block followed by a raw ABC **tune body** — just the music, no ABC header lines (`X:`, `M:`, `K:`, … are generated by the app):

```
---
title: African Market
subtitle: 2. Stimme · in B
meter: 4/4
key: C
---
B3 G z2 GG | c3/2B/2 B/2A/2G G2 z G | A3 G z2 BB/2A/2 | AG G2 z2 z G |
B3 G z2 (3GGB | B3 G z2 G/2G/2 z | z A AG c2 B3/2A/2 | z G G2 z2 z2 |]
```

Only `title`, `subtitle`, `meter`, and `key` are recognized; unknown keys are dropped on the next save. A file without a frontmatter block is read as a pure ABC body with defaults (meter `4/4`, key `C`). In the body, `%` starts a comment and a blank line ends the tune.

## Tech stack & project structure

Vue 3 (`<script setup>`) + TypeScript + Vite. No router, no state-management library — `App.vue` owns all state. abcjs 6.4.3 is loaded as a global from a CDN `<script>` tag in `index.html` (it is not an npm dependency).

```
src/
├── main.ts               entry point
├── App.vue               orchestrator: top bar, file CRUD, autosave, SVG export
├── styles.css            :root design tokens + body reset
├── env.d.ts              hand-written types for ABCJS and the File System Access API
├── components/
│   ├── Editor.vue        split view, toolbar, ABC rendering, print fixups
│   └── Sidebar.vue       offcanvas file list + folder picker
└── lib/
    ├── abc.ts            generates the ABC tune header
    ├── frontmatter.ts    .notation parse/serialize + filename slugify
    ├── storage.ts        File System Access API + IndexedDB handle persistence
    └── types.ts          NotationDoc and friends
```
